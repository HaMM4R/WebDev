Critical Log

Interoperability

Interoperability is an important thing to keep in mind when developing a website. It was important that the site be useable on both mobile and desktop devices with minimal issue. It was decided that using a CSS grid system would be the best way to achieve this, with elements taking up a different number of columns based on the resolution of the display. This was achieved using media queries and (as seen in the browser compatibility screenshots) results in elements being shifted in to accommodate mobile devices. Another example of this would be the navigation bar changing to a dropdown menu when the resolution drops too low. This was important so that users could make use of the site on any device. The website has also been tested on multiple browsers, including Google Chrome, Mozilla Firefox, Internet Explorer and Microsoft Edge. All functionality of the website works on these browsers, however an issue between browsers occurred when using a filter to invert the colours on the Brian May logo on internet explorer. Using vendor specific prefixes was attempted to correct this issue however no solution was found.

Challenges Presented

Originally when attempting to make the website responsive, a CSS grid was not implemented. This made the positioning of elements within the page very awkward. Implementing CSS grid made positioning these elements much easier.

Techniques Used

Various techniques have been applied to ensure the website is as functional as possible. For example, the use of JavaScript for things like modal boxes, JQuery for drag and drop functionality for the customiser and local storage to save a canvas ready for being loaded. JSON was used for this in order to stringify two arrays (for X and Y positions) of the user dragged and drop “Greatest Hits” text.

Strengths and Weaknesses of Current Web Standards

Newer web standards allow for content that would have not been possible before. An example of this would be the addition of canvas elements in HTML5 which were not present in HTML4. This was greatly important to the development of the site as it allowed for the creation of the customiser. This kind of functionality would have been very difficult to achieve using HTML4, especially without the use of external libraries. HTML5 semantic tags such as header, footer and nav have made it easier for developers to interpret the code written for a page. It helps to outline what sections of the mark-up are used for and helps with searchability and reduces the amount of div’s required for a website. Updating the web standards means that certain tags and attributes are no longer valid. This could become an issue for developers as it means they will have to invest time and resources into changing their websites for the new standards. For example center tags have been removed and elements now require to be centered through CSS.

Following web standards in general does have it’s merits, interoperability being a large one, as well as making it easy for developers to understand other developers code, however it does mean that developers must pay much more attention to how they are programming their websites, potentially taking more time for development and requiring more skill to develop validated websites.

Best Practises

Upholding good practises is essential in ensuring the website will be future proof and easy to maintain. Validating the html and CSS of a website will increase the chances of a website will work on future versions of browser (as they will be designed with the W3C web standards in mind) and by validating the code for the website these standards are met. This will also greatly increase the chances of successful interoperability between various browsers. Due to these standards being understood as best practise by most developers, programming a website with these standards in mind make it easier for other developers to understand the website code. Things such as the structure of the html was also kept in mind to make the code easy to read, as well as keeping the file structure clear and logical. Most of the issues flagged by W3C were for using buttons to aid in navigation instead of using tags. This was done mostly for the purpose of easier styling. Making sure that the code is consistent is also greatly important, this was achieved by typing variable names using camelCase and method names using PascalCase. The website is also compatible with most popular browsers, mostly without issue. In terms of design the website is consistent throughout, using google fonts and following design choices made by many popular websites, things like including the logo in the top left of the navigation bar.
